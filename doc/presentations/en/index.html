A bit confusing right?

UI testing should not



Be a way to implement the server APIs test!


If there is a change in the API and in the UI at the same time, troubleshooting becomes complex

UI testing should not



Rely on one's memories


The expected result of a UI test cannot be based on a memory

UI testing should be



A way to discover UI problems


Detecting broken links, wrong button position or even bad phone numbers!

UI testing should be



Automated


Checking all links and buttons styles, text and positions is not impossible!

UI testing should be



Executed before testers download the product


Testers you have to prove your brain's value by doing things that cannot be automated!
